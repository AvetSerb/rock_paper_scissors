# RPS Telnet Server — мультиплеер “Камень-Ножницы-Бумага” (Java)

Небольшой TCP-сервер для игры в “Камень-Ножницы-Бумага”.
Игроки подключаются по `telnet`/`nc`, вводят ник, сервер подбирает оппонента и запускает бой.
Игроки вводят `rock/paper/scissors` (также можно по-русски: `камень/ножницы/бумага`).
При ничьей раунд повторяется. При победе одного из игроков сессия завершается и соединения закрываются.

## Требования

- macOS
- JDK 17+
- (Опционально) `telnet`. На macOS его по default нет — используем `nc` (netcat).

Проверить Java:

```bash
    java -version
        Сборка (macOS / Linux)
        
    Из корня проекта:
        ./build.sh
        
Запуск сервера
    По умолчанию сервер слушает порт 2323:
        ./run.sh
        
    Запуск на другом порту (например 5555):
        ./run.sh 5555
        
    После запуска в консоли появится лог вида:
    RPS server listening on 0.0.0.0:2323
    
Как сыграть (2 клиента)

Откройте два окна терминала.
    Вариант 1: через nc (рекомендуется на macOS)
        Терминал A:
            nc localhost 2323
                Введите ник и нажмите Enter (например artem).
                
         Терминал B:
            nc localhost 2323
                Введите ник (например bob).
                    После того как сервер найдёт пару, введите ход:
                        rock / paper / scissors или камень / бумага / ножницы
                        
Если вы подключили только одного клиента — он будет ждать второго игрока.

    Вариант 2: через telnet (если установлен)
        telnet localhost 2323
        
Правила: Ничья → раунд повторяется.
Есть победитель → сервер отправляет результат обоим игрокам и закрывает соединения (ПО ТЗ).

Полезные команды для диагностики на macOS
    Проверить, что порт слушается:
        lsof -nP -iTCP:2323 -sTCP:LISTEN
    Проверить доступность порта:
        nc -vz localhost 2323
    Узнать IP мака (если хотите подключиться с другого устройства в сети):
        ipconfig getifaddr en0
    Подключение с другого устройства:
        nc <IP_вашего_мака> 2323
        
        
        
Архитектура:
Main — точка входа, читает порт, запускает сервер.
Server — принимает подключения (accept) и создаёт обработчики клиентов.
ConnectionHandler — спрашивает ник и отправляет игрока в лобби.
MatchMaker — подбирает пары игроков из очереди.
GameSession — ведёт матч: запрашивает ходы, определяет победителя, обрабатывает ничьи, завершает соединения.
Move — парсинг и сравнение ходов (rock/paper/scissors).
